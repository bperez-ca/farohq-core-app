## 3. PRODUCT SCOPE DECISIONS

### 3.1 Listings/Presence Sync

**Question:** Should we build full listing sync like Vendasta (50+ directories)?

**Decision:** **PHASED APPROACH**

| Phase | Scope | Rationale |
|-------|-------|-----------|
| MVP | GBP only | 80% of value, lowest complexity |
| Phase 2 | + Meta (FB, IG) | Already building for messaging |
| Phase 3 | + Apple, Bing, Yelp | Top 6 covers 95% of searches |
| Phase 4 | + 20 industry directories | Competitive parity |
| Year 2 | Data aggregators | Long-term accuracy |

**Why Not Full Sync in MVP:**
1. API complexity (each platform = 2-4 weeks)
2. Cost (Yext/similar APIs are expensive)
3. Focus (messaging is our core differentiator)
4. MVP validation (prove the model first)

**Recommendation:** Partner with a listings API provider (Yext, Uberall, or similar) in Phase 4 rather than building from scratch.

### 3.2 Keyword Tracking

**Question:** Should we build local keyword ranking like Vendasta?

**Decision:** **PHASE 3 (After core product validated)**

| Feature | Priority | Rationale |
|---------|----------|-----------|
| Basic SEO score in Snapshot | MVP | Quick win for sales |
| Single keyword tracking | Phase 2 | Low complexity |
| Multi-keyword (15) tracking | Phase 3 | Competitive feature |
| Local grid map | Phase 4 | Nice to have |

**Why:**
- Keyword tracking requires SERP API ($0.01-0.05/query)
- Need scale to justify the cost
- Not core to "Conversations = ROI" thesis

### 3.3 Review Management

**Question:** How does our review management compare to Vendasta Premium?

**FARO vs. Vendasta Review Features:**

| Feature | Vendasta Premium | FARO (Planned) |
|---------|------------------|----------------|
| Review aggregation | ✅ Google, FB, Yelp, 20+ | ✅ Google, FB, Yelp (Phase 2) |
| Review inbox | ✅ | ✅ MVP |
| AI-suggested responses | ✅ | ✅ Phase 2 |
| Auto-respond (AI) | ✅ | ✅ Phase 3 |
| Review requests | ✅ Email/SMS | ✅ + WhatsApp (differentiator!) |
| Sentiment analysis | ✅ | ✅ Phase 3 |
| Conversational insights | ✅ (new) | ✅ Phase 3 |

**FARO's Differentiator:** WhatsApp review requests!

In LATAM, SMS open rates are ~20%. WhatsApp open rates are ~98%.

```
FARO Review Request Flow:

1. Client completes job
2. Agency sends review request via WhatsApp
3. Customer receives friendly message with direct link
4. Customer leaves review
5. AI responds to review automatically
6. Review shows up in client dashboard

Conversion rate: 30-40% (vs 5-10% for email)
```

### 3.4 What We DON'T Build

| Feature | Vendasta Has | FARO Decision | Reason |
|---------|--------------|---------------|--------|
| Citation Builder | ✅ | ❌ Skip | US-only, complex, low ROI |
| Data aggregator sync | ✅ | ❌ Phase 4+ | Expensive, not urgent |

## 5. PRESENCE SYNC STRATEGY

### 5.1 MVP: GBP-First Approach

**Why GBP First:**
- 90%+ of local searches go through Google
- GBP is free and well-documented API
- Most agencies already have GBP access
- Proves the model before expanding

**MVP Implementation:**

```
GBP Integration (MVP):

READ OPERATIONS:
├── Business name, address, phone
├── Hours (regular + special)
├── Categories
├── Photos
├── Reviews (for aggregation)
└── Q&A

WRITE OPERATIONS:
├── Update business info
├── Update hours
├── Respond to reviews
├── Post updates (Phase 2)
└── Answer Q&A (Phase 2)

NOT IN MVP:
├── Auto-sync (manual trigger only)
├── Multi-location
└── Insights/analytics
```

### 5.2 Phase 2-3: Expand to Top 6

```
Platform Priority:

1. Google Business Profile ← MVP
2. Facebook ← Phase 2 (already building for Messenger)
3. Instagram ← Phase 2 (shared Meta API)
4. Apple Maps ← Phase 2 (growing importance)
5. Bing Places ← Phase 2 (easy API)
6. Yelp ← Phase 3 (API limitations, but important)

Coverage: These 6 platforms cover 95%+ of local searches
```

### 5.3 Phase 4+: Partner for Extended Sync

**Build vs. Partner Decision:**

| Approach | Pros | Cons |
|----------|------|------|
| Build in-house | Full control, no fees | 6-12 months, ongoing maintenance |
| Partner (Yext, Uberall) | Instant access to 50+ | Per-listing fees, dependency |
| Hybrid | Core 6 in-house, extended via partner | Complexity |

**Recommendation:** Hybrid approach
- Build GBP, Meta, Apple, Bing in-house (control)
- Partner for 20+ extended directories (speed)
- Evaluate in-house build for Year 2 based on volume

## 6. REVIEW MANAGEMENT STRATEGY

### 6.1 The WhatsApp Advantage

**Traditional Review Request Flow:**
```
Email sent → 20% open rate → 5% click → 2% leave review
Result: 2 reviews per 100 requests
```

**FARO WhatsApp Review Request Flow:**
```
WhatsApp sent → 98% open rate → 40% click → 30% leave review
Result: 30 reviews per 100 requests (15x better!)
```

**Implementation:**

```
WhatsApp Review Request:

TRIGGER OPTIONS:
├── Manual (from conversation)
├── After appointment (calendar integration)
├── After invoice (accounting integration)
└── Bulk campaign

MESSAGE TEMPLATE:
"Hi [Name]! Thanks for visiting [Business]. 
We'd love to hear about your experience. 
Would you mind leaving us a quick review? 
[One-tap Google Review Link]"

FOLLOW-UP (if no action in 24h):
"Hi [Name], just a friendly reminder 
about leaving a review. It really helps 
small businesses like [Business]! [Link]"

TRACKING:
├── Sent count
├── Delivered count
├── Clicked count
├── Review received
└── Review rating
```

### 6.2 AI Response Strategy

**Phase 2: AI-Suggested Responses**
```
User sees review → Clicks "Suggest Response" → 
AI generates 3 options (Formal, Friendly, Brief) → 
User selects/edits → Posts
```

**Phase 3: AI Auto-Responses (Configurable)**
```
Configuration Options:

AUTOMATION RULES:
├── Auto-respond to 5-star reviews: [ON/OFF]
├── Auto-respond to 4-star reviews: [ON/OFF]
├── Auto-respond to 3-star reviews: [ON/OFF]
├── Flag 2-star for human review: [ON/OFF]
├── Flag 1-star for human review: [ON/OFF]
└── Delay before responding: [0-24 hours]

RESPONSE STYLE:
├── Tone: Professional / Friendly / Casual
├── Length: Brief / Medium / Detailed
├── Include owner name: [ON/OFF]
├── Include business name: [ON/OFF]
└── Custom instructions: [text field]

KNOWLEDGE BASE:
├── Business policies
├── Common FAQ answers
├── Keywords to include (SEO)
└── Topics to avoid
```
